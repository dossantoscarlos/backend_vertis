# PHPCS Rules

- Follow PSR-12-compatible formatting.
- Keep imports ordered and remove unused imports.
- Use braces for all control structures.
- Keep lines readable; split long expressions instead of compressing code.
- Prefer expressive names over abbreviations.
- Avoid inline comments unless the logic is genuinely non-obvious.
- Keep arrays and argument lists consistently formatted.
- Ensure files remain lint-friendly and formatter-friendly.
