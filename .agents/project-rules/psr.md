# PSR Rules

- Follow PSR-1 and PSR-12 for PHP source code.
- Use `declare(strict_types=1);` only if the codebase already standardizes on it.
- Keep namespaces aligned with the directory structure.
- Use one class per file.
- Use PascalCase for classes, methods, enums, and components where applicable.
- Use camelCase for variables, parameters, and methods.
- Keep file encodings UTF-8 and line endings consistent with the repository.
