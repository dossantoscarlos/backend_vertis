# Larastan Rules

- Add explicit types to method parameters and return values.
- Use accurate PHPDoc for array shapes and complex collections.
- Avoid dynamic properties and untyped mixed values when a concrete type is known.
- Prefer narrow return types over broad unions.
- Keep collection transformations type-safe and predictable.
- Use nullable types only when null is a valid domain value.
- Update annotations when model attributes, relationships, or query results change.
